# gitdex

Turn your GitHub repos into a single, compact, LLM-ready summary.

`gitdex` scrapes every repository you own, builds a depth-capped file tree for each one, generates a one-line AI summary using a locally-run LLM (via [Ollama](https://ollama.com)), and writes everything into a single text file — perfect for pasting into an LLM context window, a portfolio doc, or a resume-writing session.

---

## Features

- 🔐 **GitHub OAuth device flow** — one-time browser login, token cached locally at `~/.github_repo_summarizer/token`, with automatic re-auth if the cached token goes stale.
- 🌳 **Smart file tree scraping** — recursively walks each repo (depth-capped at 2 levels by default) while skipping noise: `node_modules`, `.git`, build artifacts, binaries, images, fonts, lockfiles, and more.
- 🤖 **Local AI summaries** — each repo gets a short, one-line description generated by a local Ollama model (`qwen2.5:0.5b`), so nothing about your code is sent to a third-party API.
- 🎯 **Selective or full scan** — summarize everything you own, or interactively pick specific repos via checkboxes.
- 🔢 **Token counting** — check how many tokens your summary file contains (via `tiktoken`), and break that count down per repo.
- 🎨 **Rich terminal UI** — clean, formatted output powered by [Rich](https://github.com/Textualize/rich).

---

## Prerequisites

- **Python 3.9+**
- **[Ollama](https://ollama.com)** installed and running locally, with the `qwen2.5:0.5b` model pulled:
  ```bash
  ollama pull qwen2.5:0.5b
  ```
  `gitdex` calls `http://localhost:11434` to generate summaries, so Ollama must be running before you use `complete` or `srepo`.
- A **GitHub account** (used for the one-time device-flow login).

### Windows (PowerShell as Administrator)


After installation, verify everything is working:

```bash
gitdex test
```

You should see `Working!` printed to the console.

## Usage

### First run — authentication

The first time you run any command that needs GitHub access (`complete` or `srepo`), `gitdex` will open your browser for a one-time device login and cache the token locally. You won't need to log in again unless the token is revoked or expires.

### Commands

| Command | Arguments | What it does |
|---|---|---|
| `complete` | `<output.txt>` | Authenticates, scrapes **every** repo you own, and writes a full summary + file tree to a text file. |
| `srepo` | `<output.txt>` | Same as `complete`, but lets you **interactively pick** which repos to include via checkboxes. |
| `count_token` | `<file.txt>` | Counts how many tokens a text file contains (`tiktoken`, `cl100k_base` / GPT-4 encoding). |
| `token_by_repo` | `<summary.txt>` | Takes a summary file (from `complete`/`srepo`) and prints a per-repo token breakdown. |
| `test` | — | Quick sanity check that `gitdex` is installed and runnable. |
| `help` | — | Shows a full, formatted guide of every command in-terminal. |

### Examples

```bash
# Summarize every repo you own into output.txt
gitdex complete output.txt

# Interactively choose which repos to summarize
gitdex srepo output.txt

# Check total token count of a summary file
gitdex count_token output.txt

# See a per-repo token breakdown
gitdex token_by_repo output.txt

# Full command reference, right in your terminal
gitdex help
```

## How it works

1. **Auth** — `gitdex` performs a GitHub device-flow login, caching your access token locally (`chmod 600`) so you're only prompted once.
2. **Scrape** — for each repo, it walks the file tree up to a configurable depth, filtering out build artifacts, dependency folders, binaries, and other non-informative files/extensions.
3. **Summarize** — the resulting tree (plus the repo's GitHub description, if available) is passed to a local Ollama model, which returns a single-sentence summary — no code or repo data leaves your machine for this step beyond the local Ollama call.
4. **Write** — repo name, file tree, and summary are appended to your output file, one block per repo.
5. **Analyze** — use `count_token` or `token_by_repo` to understand how large your summary file is, so you can trim it before feeding it into an LLM's context window.

## Output format

Each repo in the output file looks like this:

```
Repo Name: gitdex
gitdex
 __init__.py
 cli.py
 mainFile.py
install.ps1
install.sh
pyproject.toml
Summary: A CLI tool that scrapes GitHub repos and generates AI summaries using a local LLM.

```

---

## Notes

- Ignored directories include (but aren't limited to): `.git`, `node_modules`, `dist`, `build`, `__pycache__`, `.venv`, `target`, `.idea`, platform build folders (`android`, `ios`, `linux`, etc.), and IDE/cache folders.
- Ignored file types include: compiled binaries (`.exe`, `.dll`, `.so`, `.pyc`...), media (`.jpg`, `.mp4`, `.mp3`...), fonts, lockfiles, and minified/CSS/SVG/Markdown files.
- The default tree depth is capped at 2 levels to keep summaries compact — deeper nesting is skipped rather than fully expanded.
